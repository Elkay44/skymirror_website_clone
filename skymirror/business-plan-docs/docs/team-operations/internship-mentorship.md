# Mentorship & Evaluation Framework

## Overview

This framework defines the mentorship model, evaluation criteria, and feedback mechanisms for the Skymirror internship program. It ensures consistent, high-quality mentorship and fair, transparent evaluation of intern performance.

**Audience:** Mentors, Interns, Program Lead  
**Last Updated:** December 2024

- [Roadmap](./internship-roadmap.md) - Week-by-week plan
- [Playbook](./internship-playbook.md) - Daily operations
:::

## Mentorship Model

### Philosophy

**Our Mentorship Approach:**
- **Guide, don't solve** - Help interns find answers, don't give them
- **Teach to fish** - Build problem-solving skills, not dependency
- **Safe to fail** - Create environment where mistakes are learning opportunities
- **Growth mindset** - Focus on progress, not perfection
- **Individual attention** - Each intern is unique with different needs

### Mentor-Intern Ratio

**1:1 Assignment**
- Each intern has one primary mentor
- Mentor is a senior engineer (3+ years experience)
- Backup mentor assigned for coverage
- Access to entire team for questions

## Mentor Responsibilities

### Time Commitment

| Activity | Frequency | Duration | Total/Week |
|----------|-----------|----------|------------|
| **1:1 Meeting** | Weekly | 60 min | 1 hour |
| **Code Reviews** | Daily | 30-60 min | 3-5 hours |
| **Ad-hoc Questions** | Daily | 15-30 min | 1-2 hours |
| **Sprint Planning** | Weekly | 30 min | 0.5 hours |
| **Program Meetings** | Weekly | 30 min | 0.5 hours |
| **Total** | - | - | **6-9 hours/week** |

### Core Responsibilities

#### 1. Technical Guidance (40%)

**Code Reviews:**
- Review all intern PRs within 24 hours
- Provide constructive, educational feedback
- Explain the "why" behind suggestions
- Approve when ready (don't nitpick)

**Technical Support:**
- Answer questions (response time: under 2-hours)
- Help debug complex issues
- Explain architectural decisions
- Share best practices

**Pair Programming:**
- At least 2 sessions per week (Week 1-2)
- Demonstrate problem-solving approach
- Think out loud to model thought process

#### 2. Task Management (20%)

**Sprint Planning:**
- Assign appropriate tasks
- Ensure clear requirements
- Set realistic expectations
- Break down complex work

**Progress Monitoring:**
- Daily check-ins via Slack
- Review Jira updates
- Identify blockers early
- Adjust workload as needed

#### 3. Career Development (20%)

**Skill Building:**
- Identify learning opportunities
- Recommend resources
- Provide stretch assignments
- Share industry insights

**Feedback & Coaching:**
- Weekly 1:1 discussions
- Constructive feedback
- Celebrate wins
- Address concerns

#### 4. Program Support (20%)

**Documentation:**
- Weekly progress reports
- Evaluation forms
- Incident reports (if needed)

**Collaboration:**
- Attend mentor meetings
- Share best practices
- Support other mentors
- Contribute to program improvement

## Evaluation Schedule

### Continuous Evaluation

**Daily:**
- Code review feedback
- Stand-up participation
- Slack communication
- Task updates

**Weekly:**
- 1:1 discussion
- Sprint performance
- Learning progress
- Informal feedback

### Formal Evaluations

#### Week 2: Mid-Program Review

**Purpose:** Course correction and support

**Format:**
- 30-minute meeting
- Review progress against goals
- Identify challenges
- Adjust plan if needed
- No formal rating

**Discussion Points:**
- What's going well?
- What's challenging?
- Do you feel supported?
- What can we adjust?
- Are goals still appropriate?

#### Week 4: Final Evaluation

**Purpose:** Comprehensive assessment and next steps

**Format:**
- 60-minute meeting
- Complete evaluation rubric
- Overall rating (1-5)
- Written feedback
- Career discussion

**Components:**
1. **Self-Assessment** (intern completes first)
2. **Mentor Evaluation** (using rubric)
3. **Discussion** (compare and discuss)
4. **Next Steps** (career planning)

## Feedback Best Practices

### Giving Effective Feedback

#### The SBI Model

**Situation - Behavior - Impact**

**Example:**
```
Situation: "In yesterday's code review..."
Behavior: "...you explained your architectural decision clearly..."
Impact: "...which helped the team understand the trade-offs and approve the PR quickly."
```

#### Positive Feedback

**Do:**
- Be specific and timely
- Explain why it matters
- Encourage repetition
- Celebrate publicly (if appropriate)

**Example:**
```
âœ… "Your PR description for the auth feature was excellent. You included:
   - Clear problem statement
   - Implementation approach
   - Testing steps
   - Screenshots
   
   This made review much faster and shows great communication skills.
   Keep doing this!"
```

#### Constructive Feedback

**Do:**
- Focus on behavior, not person
- Be specific with examples
- Suggest improvements
- Offer support

**Don't:**
- Be vague ("your code needs work")
- Make it personal ("you're not good at...")
- Only criticize without solutions
- Give feedback publicly (unless minor)

**Example:**
```
âœ… "I noticed in the last few PRs, some edge cases weren't handled:
   - Null checks missing in user.service.ts
   - Empty array not handled in getUsers()
   
   This can cause production bugs. Let's pair program tomorrow to
   review defensive programming patterns. I'll share some resources too."
```

## Difficult Conversations

### When Performance is Below Expectations

#### Early Intervention (Week 1-2)

**If intern is struggling:**

1. **Identify the issue**
   - Technical skills gap?
   - Time management?
   - Communication?
   - Personal issues?

2. **Have honest conversation**
   - "I've noticed [specific behavior]"
   - "This is concerning because [impact]"
   - "How can I support you?"

3. **Create action plan**
   - Specific improvements needed
   - Timeline for improvement
   - Additional support offered
   - Check-in schedule

4. **Document conversation**
   - Date and summary
   - Action items agreed
   - Follow-up date

#### Mid-Program Intervention (Week 2-3)

**If issues persist:**

1. **Escalate to Program Lead**
   - Share documentation
   - Discuss options
   - Get guidance

2. **Formal improvement plan**
   - Written expectations
   - Clear timeline
   - Daily check-ins
   - Possible outcomes

3. **Consider alternatives**
   - Extended internship?
   - Different role/project?
   - Early conclusion?

### Sample Difficult Conversation Script

```
"[Name], I wanted to talk with you about your progress. Over the past
two weeks, I've noticed [specific issues]:

1. [Issue 1 with example]
2. [Issue 2 with example]
3. [Issue 3 with example]

This is concerning because [impact on learning/team/project].

I want to understand what's happening from your perspective. What
challenges are you facing? How can I better support you?

[Listen to response]

Here's what I need to see improve:
- [Specific expectation 1]
- [Specific expectation 2]
- [Specific expectation 3]

Let's create a plan together. I'll [mentor support], and you'll
[intern actions]. We'll check in daily for the next week.

Does this sound fair? Do you have questions or concerns?"
```

## FAQ for Mentors

**Q: How much time should I spend mentoring?**
A: 6-9 hours per week on average. More in Week 1-2, less in Week 3-4 as intern becomes independent.

**Q: What if my intern is struggling?**
A: Intervene early. Have honest conversation, create action plan, escalate to Program Lead if needed.

**Q: How strict should I be in code reviews?**
A: Balance learning and quality. Explain the "why" behind feedback. Don't nitpick, but don't approve bad code.

**Q: What if I'm too busy to mentor well?**
A: Communicate early. We can adjust workload or assign backup mentor. Intern success is priority.

**Q: How do I give negative feedback?**
A: Use SBI model. Be specific, kind, and constructive. Focus on behavior, not person. Offer solutions.

**Q: What if intern asks questions I don't know?**
A: It's okay to say "I don't know, let's figure it out together." Models problem-solving.

**Q: Should I do work for the intern?**
A: No. Guide them to the solution. Pair program if needed, but they should drive.

**Q: How do I balance being friendly and professional?**
A: Be approachable and supportive, but maintain professional boundaries. You're mentor first, friend second.
