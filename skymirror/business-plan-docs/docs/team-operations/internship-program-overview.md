# SkyMirror Internship Program

## Executive Overview

The SkyMirror Internship Program is a comprehensive 6-month professional development initiative designed to transform talented individuals into production-ready software engineers and future Academy instructors. Our program combines hands-on product development, structured learning, mentorship, and teaching opportunities to deliver real business value while building long-term career paths within SkyMirror.

**Program Lead:** CTO (Eric)  
**Scope:** Engineering internships across all product lines  
**Duration:** 6 months (extendable to 12 months)  
**Last Updated:** December 2024

## Program Philosophy

### Our Approach
We believe in **learning by building**. Interns don't work on toy projectsâ€”they ship real features to production, contribute to actual products, and experience the full software development lifecycle.

### Core Principles
1. **Real Impact** - Every intern ships production code
2. **Structured Learning** - Balanced curriculum with hands-on application
3. **Professional Standards** - Fortune 50-level processes and quality
4. **Mentorship First** - Every intern has a dedicated technical mentor
5. **Growth Mindset** - Focus on learning, not perfection

### Tool Usage Guidelines

**Trello** - Personal Productivity
- Update daily (morning planning, end-of-day review)
- Break tasks into <2 hour chunks
- Move cards through workflow honestly

**Jira** - Professional Work
- Update status in real-time
- Log time spent (if tracking enabled)
- Link commits and PRs to tickets
- Comment on blockers immediately

**Google Calendar** - Time Discipline
- Block all fixed commitments
- Respect others' calendar blocks
- Update if plans change

**Slack** - Communication
- Response time: <2 hours during work hours
- Use threads for discussions
- Ask questions in public channels (helps everyone learn)

## Eligibility & Selection

### Ideal Candidate Profile

**Technical Foundation**
- Basic programming knowledge (any language)
- Understanding of web fundamentals (HTTP, APIs, databases)
- Git basics (or willingness to learn quickly)
- Problem-solving aptitude

**Professional Qualities**
- Strong communication skills
- Self-directed learning ability
- Growth mindset
- Team collaboration
- Time management discipline

**Commitment**
- Full-time availability (40 hours/week)
- Dedicated workspace and reliable internet
- Willingness to work in fast-paced environment

### Selection Process

1. **Application Review** (1 week)
   - Resume and portfolio review
   - GitHub profile assessment
   - Cover letter evaluation

2. **Technical Assessment** (1 week)
   - Coding challenge (3-4 hours)
   - Problem-solving evaluation
   - Code quality review

3. **Interviews** (1 week)
   - Technical interview (1 hour)
   - Behavioral interview (30 min)
   - Culture fit assessment

4. **Offer & Onboarding** (1 week)
   - Offer letter
   - Pre-arrival materials
   - Tool access setup

## Risk Management

### Common Challenges & Mitigation

| Risk | Mitigation Strategy |
|------|---------------------|
| **Intern overwhelm** | Clear task scoping, regular check-ins, mental health support |
| **Technical blockers** | Mentor availability, documentation, pair programming |
| **Scope creep** | Strict sprint planning, mentor oversight, task sizing |
| **Communication gaps** | Daily stand-ups, Slack guidelines, escalation path |
| **Quality issues** | Code review process, testing requirements, mentor oversight |
| **Time management** | Calendar blocking, time tracking, workload monitoring |

### Red Flags & Interventions

**Warning Signs:**
- Missing stand-ups or updates
- No Jira progress for 2+ days
- Declining code quality
- Lack of questions (too quiet)
- Excessive overtime

**Intervention Protocol:**
1. Mentor 1:1 conversation
2. Adjust workload if needed
3. Additional support resources
4. Team lead involvement
5. Program lead escalation (if needed)
